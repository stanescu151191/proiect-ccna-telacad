Scenarii de Testare - Proiect CCNA

SCENARIU 1 - Failover HSRP

Scop: Verificarea redundantei gateway-ului implicit.

Descriere: Reteaua 1 foloseste protocolul HSRP pentru a asigura redundanta gateway-ului. sw-dist01 este Active pentru VLAN 10, 20, 40 iar sw-dist02 este Active pentru VLAN 50, 77, 99 - realizand load balancing.

Pasi:

1. Verificam starea initiala: show standby brief

2. Pornim ping continuu de pe Conta1 spre 10.30.99.10

3. Oprim interfata Gi0/1 pe sw-dist01

4. sw-dist02 preia rolul Active pe toate VLANurile

5. Repornm sw-dist01 - recapata rolul Active (preempt)

Rezultat: Pierdere maxim 2-3 pachete in momentul failover.

Concluzie: Reteaua asigura redundanta completa a gateway-ului fara interventie manuala.

SCENARIU 2 - Failover STP

Scop: Verificarea redundantei la nivel de acces.

Descriere: Fiecare switch de acces are doua uplink-uri spre switch-urile de distributie. STP blocheaza unul dintre ele pentru a evita loop-urile. In caz de defectiune, portul blocat devine activ automat.

Pasi:

1. Verificam portul blocat: show spanning-tree vlan 10

2. Fa0/24 pe sw-acc01 = Altn BLK (blocat de STP)

3. Deconectam fizic cablul Fa0/23 (uplink primar)

4. STP deblocheaza Fa0/24 dupa ~30 secunde

5. Conectivitatea se restabileste automat

Rezultat: Convergenta STP in aproximativ 30 secunde.

Concluzie: Redundanta fizica la nivel de acces asigurata prin protocolul STP.

SCENARIU 3 - Rutare Redundanta R1

Scop: Verificarea redundantei linkurilor dintre SwitchBlock si Router-ul central R1.

Descriere: R1 are doua linkuri active spre Reteaua 1 - unul spre sw-dist01 (172.16.0.0/30) si unul spre sw-dist02 (172.16.0.4/30). Ambele linkuri sunt active simultan, distribuind traficul in mod automat.

Pasi:

1. Verificam rutele pe R1: show ip route

2. Vedem doua rute spre 10.10.0.0/16

3. Oprim interfata Gi0/1 pe sw-dist01

4. R1 foloseste automat doar ruta via sw-dist02

5. Conectivitatea se mentine fara intrerupere

Rezultat: Zero pierderi de pachete la failover.

Concluzie: Rutare redundanta cu distributia traficului pe doua linkuri active simultan.

SCENARIU 4 - Failover EtherChannel

Scop: Verificarea redundantei EtherChannel dintre sw-dist01 si sw-dist02.

Descriere: Intre cele doua switch-uri de distributie exista un EtherChannel LACP (Po1) format din doua linkuri fizice: Fa0/23 si Fa0/24. Daca unul dintre linkuri pica, traficul continua pe celalalt link automat.

Pasi:

1. Verificam: show etherchannel summary

2. Po1(SU) Fa0/23(P) Fa0/24(P) - ambele active

3. Oprim Fa0/23 pe sw-dist01

4. Po1 ramane SU dar doar cu Fa0/24(P)

5. Traficul continua pe Fa0/24 fara intrerupere

Rezultat: EtherChannel mentine conectivitatea pe un singur link fizic.

Concluzie: Redundanta la nivel de link intre switch-urile de distributie.
